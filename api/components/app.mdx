## Overview

The `App` component is the root component of PromoMaster Pro. It manages application state, localStorage persistence, navigation between views, and CRUD operations for stores, products, and announcements.

## Component Signature

```tsx
const App: React.FC = () => { ... }
```

The App component has no props and is self-contained.

## State Management

### AppState Interface

### Notification System

```tsx
interface Notification {
  id: string;
  message: string;
  type: 'success' | 'info' | 'error';
}
```

The component maintains a separate notification state for toast messages with auto-dismiss after 4 seconds.

## Key Features

### LocalStorage Persistence

The application automatically persists state to localStorage under the key `promomaster_data_v2`:

- State is loaded on mount from localStorage
- State is saved to localStorage on every update
- Handles parse errors gracefully with fallback to default state

## View Routing Logic

The App component conditionally renders views based on `state.view`:

### Display View

When `state.view === 'display'`, the component:

1. Resolves `DisplayItem[]` references to actual Product/Announcement objects
2. Validates that items and store exist
3. Renders the `AdDisplay` component in fullscreen
4. Falls back to dashboard if validation fails

```tsx
if (state.view === 'display') {
  return (
    <AdDisplay 
      items={resolvedItems} 
      store={store} 
      duration={state.displayDuration || 5}
      onExit={() => setView('dashboard')} 
    />
  );
}
```

### Dashboard View

```tsx
<Dashboard 
  products={state.products} 
  announcements={state.announcements}
  stores={state.stores} 
  onStartDisplay={startDisplay}
  defaultDuration={state.displayDuration || 5}
/>
```

### Management View

```tsx
<Management 
  stores={state.stores}
  products={state.products}
  announcements={state.announcements}
  onAddStore={addStore}
  onUpdateStore={updateStore}
  onDeleteStore={deleteStore}
  onAddProduct={addProduct}
  onUpdateProduct={updateProduct}
  onDeleteProduct={deleteProduct}
  onAddAnnouncement={addAnnouncement}
  onUpdateAnnouncement={updateAnnouncement}
  onDeleteAnnouncement={deleteAnnouncement}
/>
```

## Notification Management

The `addNotification` callback creates toast notifications:

```tsx
const addNotification = useCallback(
  (message: string, type: Notification['type'] = 'success') => {
    const id = crypto.randomUUID();
    setNotifications(prev => [...prev, { id, message, type }]);
    setTimeout(() => {
      setNotifications(prev => prev.filter(n => n.id !== id));
    }, 4000);
  },
  []
);
```

- Auto-generates unique IDs using `crypto.randomUUID()`
- Auto-dismisses after 4 seconds
- Supports success, info, and error types
- Renders with animated slide-in from right

## Navigation UI

The App component includes a persistent sidebar navigation:

- **Dashboard**: Main control panel with playlist builder
- **Management**: CRUD interface for stores, products, and announcements

Navigation state is managed through the `view` state property.

## Default State

If localStorage is empty or corrupted, the app initializes with:

```tsx
{
  stores: [
    { 
      id: '1', 
      name: 'Sucursal Central', 
      address: 'Av. Libertador 1234', 
      logoColor: 'bg-blue-600' 
    }
  ],
  products: [],
  announcements: [],
  displayDuration: 5,
  view: 'dashboard'
}
```
